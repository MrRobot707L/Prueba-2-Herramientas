PARTE 2: MI MODULO - ANALISIS EXPLORATORIO DE DATOS (3-4 minutos)
--------------------------------------------------------------------------------

[INTRODUCCION]

El modulo de Analisis Exploratorio de Datos que yo desarrolle permite visualizar 
y entender el dataset antes de aplicar machine learning. Esta ubicado en el 
archivo ui/analisis_datos.py.

[SECCION 1: EXPLORACION INICIAL]

Primero mostramos las estadisticas descriptivas del dataset. Aqui el usuario 
puede ver:
- El numero total de registros: 1,599 vinos
- Medias y desviaciones estandar de cada variable
- Valores minimos y maximos
- Los percentiles 25%, 50% y 75%

Tambien mostramos las primeras filas del dataset en un expander para que el 
usuario pueda ver los datos crudos.

[SECCION 2: DISTRIBUCION DE LA CALIDAD]

Luego visualizamos la distribucion de la variable objetivo usando un grafico de 
barras con Seaborn.

[MOSTRAR GRAFICO Y EXPLICAR]

Aqui podemos ver que la mayoria de los vinos tienen calidad 5 o 6, mientras que 
los vinos premium con calidad 7 u 8 son mucho menos frecuentes. Esto representa 
un problema de clases desbalanceadas que hay que tener en cuenta al entrenar el 
modelo.

[SECCION 3: TRANSFORMACION DE DATOS - DTS]

En la seccion de Transformacion de Datos mostramos como convertimos el problema 
de multiclase a clasificacion binaria.

Creamos la columna target donde:
- 1 significa Premium (calidad >= 7)
- 0 significa Estandar (calidad < 7)

Mostramos un grafico de pastel que visualiza la proporcion resultante: 
aproximadamente 13.5% de los vinos son Premium y 86.5% son Estandar.

Esta binarizacion es necesaria para que el Perceptron Multicapa pueda aprender a 
clasificar correctamente, ya que simplifica el problema y permite usar una 
funcion de activacion Sigmoid en la salida.

[SECCION 4: MATRIZ DE CORRELACION]

Finalmente, el mapa de correlacion nos permite identificar que variables 
quimicas influyen mas en la calidad.

[MOSTRAR HEATMAP Y EXPLICAR]

Las tres variables con mayor correlacion con la calidad son:
1. Alcohol: correlacion positiva (~0.48) - mas alcohol, mejor calidad
2. Acidez volatil: correlacion negativa (~-0.39) - menos acidez, mejor calidad
3. Sulfatos: correlacion positiva (~0.25) - mas sulfatos, mejor calidad

Este insight es util para el modulo de prediccion, ya que sugiere que variables 
son mas importantes al momento de ingresar datos de un vino nuevo.

--------------------------------------------------------------------------------
PARTE 3: PROCESAMIENTO DE DATOS (1-2 minutos)
--------------------------------------------------------------------------------

Tambien me encargue de generar los archivos de datos en la carpeta data/.

El archivo generar_pkl.py es un script que:
1. Carga el dataset CSV original
2. Crea la columna target para clasificacion binaria
3. Guarda el resultado como archivo pickle (.pkl)

[SI PREGUNTAN PORQUE PKL]

Respecto a porque usamos formato pkl en vez de csv:

El formato pickle es binario y permite cargar los datos mucho mas rapido, 
aproximadamente 10 veces mas rapido que un CSV. Ademas preserva exactamente los 
tipos de datos sin necesidad de parsear texto cada vez que se carga.

Esto es importante porque el modulo de prediccion necesita cargar los datos cada 
vez que el usuario entra a esa pantalla, y con pkl la carga es instantanea.
